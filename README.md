# Alfred Sales Pack

**Status:** draft v1  
**Purpose:** собрать в одном месте builder-артефакты для тёплого legal lead / first demo / post-demo follow-up.

---

## What this pack is for

Этот пакет нужен, чтобы Alfred можно было не только технически допилить, но и нормально показать живому человеку:
- коротко объяснить ценность
- снять типовые возражения
- провести первое демо
- не потерять контакт после показа

Иными словами: это не «маркетинг ради маркетинга», а operational sales rail для первого warm lead.

---

## Pack contents

### 1) One-pager
**File:** `2026-03-08-alfred-one-pager-draft-v1.md`

Что это:
- короткое позиционирование Alfred
- ценность для частнопрактикующего юриста
- ROI / time-save framing
- базовая CTA на тест реальным кейсом

Когда использовать:
- перед первым разговором
- как короткий owner/client-facing summary
- как опорный текст для переписки

---

### 2) Objection handling
**File:** `2026-03-08-alfred-objection-handling-v1.md`

Что это:
- типовые возражения
- короткие и усиленные ответы
- переходы к тесту
- anti-patterns, чтобы не продавать воздух

Когда использовать:
- в разговоре
- после первого скепсиса
- как internal rail для того, кто показывает Alfred

---

### 3) Demo script
**File:** `2026-03-08-alfred-demo-script-v1.md`

Что это:
- 5–7 минутный сценарий показа
- opening, framing, pain anchor, result framing, close
- как не превратить демо в AI-цирк

Когда использовать:
- перед live demo
- как шпаргалку для показа на одном кейсе

---

### 4) Follow-up message pack
**File:** `2026-03-08-alfred-follow-up-message-pack-v1.md`

Что это:
- follow-up ветки после демо
- interested / hesitant / no-time
- короткие ping-сообщения
- ответы на типовые follow-up вопросы

Когда использовать:
- после показа
- чтобы не потерять контакт
- чтобы возвращать разговор в один конкретный next step

---

## Recommended use order

### Minimum path
1. прочитать one-pager
2. взять objection handling
3. провести demo по script v1
4. отправить follow-up по одной из веток

### Warm lead path
1. коротко объяснить Alfred через one-pager logic
2. не спорить абстрактно, а быстро перейти к реальному кейсу
3. провести demo на типовом документе
4. закрыть на маленький тестовый шаг

---

## Positioning lock

Главная формула всего пакета:

> **Alfred не заменяет юриста. Он экономит часы на первой версии документа и разборе материалов.**

Если из этой рамки выйти, всё начинает выглядеть как хрупкая AI-демка, а не как рабочий инструмент.
